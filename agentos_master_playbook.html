1. Bootstrapping Projects

Initialize your environment correctly to ensure agents obey the governance rules from the first commit.

New Project Automation
new <project-name>

When: Starting a fresh project from scratch.

How: Clones template, creates private GitHub repo, and scaffolds.

"Why: Saves 10 minutes of manual setup and ensures 100% compliance."

Retrofit Governance
agent-os

When: Adding rules to an existing repository.

How: Injects AGENTS.md, proxies, and appends to .gitignore.

"Why: Prevents agents from messing up legacy folder structures."

The Governance Core

Bootstrapping installs the AGENTS.md file. This is the "brain" of your project. It contains strict directives that forbid committing to main and force agents to sync before starting.

3. Development Optimization

Reduce token consumption by up to 40% using structural mapping and surgical rules.

The 'Map' Directive

Always run `map` before launching an agent. This builds a tiny text tree that prevents the agent from reading every file just to understand the folder structure.

map

1. Surgical Read

Agents are forbidden from reading files over 200 lines if they only need specific blocks.

2. No Probing

Pin dependencies in code comments so agents don't waste tokens "searching" for API versions.

3. Memory Reuse

Forced to rely on session memory for files modified in the same turn.

4. The Handoff Protocol

The bridge between agents. GitHub holds the code; TODO_AGENT.md holds the memory.

Phase A: The Outbound (Leaving Agent)

Copy this to the agent you are currently using when tokens are low.

We are pausing this session and handing off to another agent. 
Perform our standard handoff protocol immediately:
1. Stage and commit all active files.
2. Push this branch to origin.
3. Completely overwrite the TODO_AGENT.md file using our master template. 
Be concise. Write a clear 'Immediate Next Step'.
Do not include any conversational summary. Execute now.

Phase B: The Inbound (Arriving Agent)

Copy this to the new agent you just opened in the folder.

We are resuming a development session. Execute these steps:
1. Run `git pull` to fetch the latest codebase state.
2. Open and read TODO_AGENT.md to capture our goal and blockers.
3. Execute the 'Immediate Next Step' listed in that file.
Do not give me any introductory conversational filler. 
Go straight to coding.

5. State Recovery

Use checkpoints for micro-pauses or unexpected task switches without writing a full handoff note.

❄️

The 'Checkpoint' Command

Freezes local uncommitted work with a 1-line hint.

checkpoint "middle of debugging odoo payload"

How to Resume

  • • Same Agent: Just say "Let's unfreeze and keep going."
  • • New Agent: Say "Check the top of TODO_AGENT.md for the checkpoint hint."

Best Practice

When the feature is finally 100% done, tell the agent:
"Squash our temporary checkpoints into one semantic commit."
